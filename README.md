# Рекомендательная система фильмов на основе MovieLens Dataset

## Описание архитектуры

Приложение построено на функциональном подходе (без ООП) и состоит из трёх независимых слоёв:

**Backend** — загрузка данных, операции над справочниками, формирование текстовых отчётов (`Library/`, `Scripts/`).

**ML** — рекомендательная система и генерация графических отчётов (`Scripts/`).

**Frontend** — графический интерфейс на Tkinter, отображение отчётов и графиков, настройки приложения (`Scripts/`).

### Жизненный цикл приложения

При первом запуске данные автоматически загружаются из архива MovieLens и преобразуются в справочники в формате 3НФ, после чего сохраняются в бинарные файлы (pickle). При повторных запусках данные загружаются напрямую из pickle-файлов. Пользователь проходит регистрацию, указывает предпочтения по жанрам и фильмам — это служит основой для персонализированных рекомендаций. Просмотренные рекомендации можно оценивать, что корректирует дальнейшие предложения системы.

## Структура каталогов

```
Work/
├── Data/       # База данных (CSV-архив и pickle-файлы справочников)
├── Graphics/   # Сохранённые графические отчёты (.png)
├── Library/    # Универсальные функции (чтение/запись файлов, утилиты)

```

## Структура базы данных

Данные приведены к третьей нормальной форме и хранятся в шести справочниках.

### Users
| Поле | Тип | Описание |
|------|-----|----------|
| `userId` | int (PK) | Уникальный идентификатор пользователя |

### Movies
| Поле | Тип | Описание |
|------|-----|----------|
| `movieId` | int (PK) | Уникальный идентификатор фильма |
| `movieName` | str | Название фильма |

### Genres
| Поле | Тип | Описание |
|------|-----|----------|
| `genreId` | int (PK) | Уникальный идентификатор жанра |
| `genreName` | str | Название жанра |

### Movie-Genre
| Поле | Тип | Описание |
|------|-----|----------|
| `movieId` | int (FK → Movies) | Идентификатор фильма |
| `genreId` | int (FK → Genres) | Идентификатор жанра |

### Ratings
| Поле | Тип | Описание |
|------|-----|----------|
| `ratingId` | int (PK) | Уникальный идентификатор оценки |
| `userId` | int (FK → Users) | Идентификатор пользователя |
| `movieId` | int (FK → Movies) | Идентификатор фильма |
| `rating` | float | Оценка фильма (0–5) |

### Tags
| Поле | Тип | Описание |
|------|-----|----------|
| `tagId` | int (PK) | Уникальный идентификатор тега |
| `userId` | int (FK → Users) | Идентификатор пользователя |
| `movieId` | int (FK → Movies) | Идентификатор фильма |
| `tag` | str | Текстовый тег |

## Описание модулей

### Library/ — универсальные функции

Функции общего назначения, не привязанные к предметной области. Могут переиспользоваться в других проектах.

| Функция | Описание |
|---------|----------|
| `read_csv(path)` | Чтение CSV-файла, возврат DataFrame |
| `write_csv(df, path)` | Запись DataFrame в CSV |
| `save_pickle(df, path)` | Сохранение DataFrame в бинарный файл |
| `load_pickle(path)` | Загрузка DataFrame из бинарного файла |
| `write_txt(text, path)` | Запись текстового отчёта в файл |

---

### Scripts/ — специализированные модули

#### Модуль данных (Backend)

| Функция | Входные данные | Возвращает | Описание |
|---------|---------------|------------|----------|
| `load_data_csv(filepath)` | `filepath: str` | `dict` с сырыми DataFrame | Читает CSV-файлы из `Data/` |
| `convert_to_3nf(raw)` | `raw: dict` | `db: dict` с 6 таблицами в 3НФ | Преобразует сырые данные в нормализованную БД |
| `init_db(path)` | `path: str` | `db: dict` | Загружает БД из pickle или инициализирует из CSV при первом запуске |
| `save_db(db, path)` | `db: dict`, `path: str` | `None` | Сохраняет все таблицы в pickle-файлы |
| `load_db(path)` | `path: str` | `db: dict` | Загружает все таблицы из pickle-файлов |
| `add_user(db)` | `db: dict` | `int` (новый userId) | Добавляет нового пользователя |
| `delete_user(db, user_id)` | `db: dict`, `user_id: int` | `bool` | Удаляет пользователя и связанные записи |
| `add_movie(db, title, genres)` | `db: dict`, данные фильма | `int` (новый movieId) | Добавляет новый фильм |
| `delete_movie(db, movie_id)` | `db: dict`, `movie_id: int` | `bool` | Удаляет фильм и связанные записи |
| `edit_movie(db, movie_id, ...)` | `db: dict`, новые данные | `bool` | Редактирует данные фильма |

Текстовые отчёты, формируемые модулем:

| Отчёт | Описание |
|-------|----------|
| Простой отчёт | Выборка строк и столбцов из справочника |
| Статистический отчёт | Для количественных атрибутов — max, min, mean, var, std; для качественных — таблица частот |
| Сводная таблица | `pivot_table` с агрегацией, например средний рейтинг по жанру |

#### Модуль рекомендаций и графиков (ML)

| Функция | Описание |
|---------|----------|
| `get_recommendations(db, user_id, n)` | Возвращает список из `n` наиболее подходящих фильмов для пользователя |
| `filter_movies(db, genres, min_rating)` | Фильтрует фильмы по жанрам и минимальному среднему рейтингу |

Графические отчёты, сохраняемые в `Graphics/`:

| График | Описание |
|--------|----------|
| Столбчатая диаграмма | Сравнение пары качественных атрибутов |
| Гистограмма | Распределение количественного атрибута по категориям |
| Диаграмма Бокса-Вискера | Разброс количественного атрибута по категориям |
| Диаграмма рассеивания | Зависимость двух количественных атрибутов с цветовым разбиением по категории |

#### Конфигурация (Scripts/config.ini)

Файл `.ini` хранит настройки интерфейса: цвета, шрифты, пути к каталогам. Загружается при старте приложения.

```ini
[theme]
background = #ffffff
font_family = Arial
font_size = 12

[paths]
data = Work/Data/
output = Work/Output/
graphics = Work/Graphics/
```

## Зависимости

Все зависимости перечислены в файле `requirements.txt`. Для установки выполните:

```bash
pip install -r requirements.txt
```

Приложение разработано для **Python 3.X (CPython)** в составе дистрибутива **Anaconda**. Используемые библиотеки: `pandas`, `numpy`, `matplotlib`, `tkinter` (входит в стандартную библиотеку).

## Запуск приложения

```bash
python Scripts/main.py
```

При первом запуске убедитесь, что в папке `Data/` находится архив с датасетом MovieLens. Приложение автоматически распакует архив, сформирует справочники и сохранит их в бинарном формате для последующих запусков.
