# Рекомендательная система фильмов на основе MovieLens Dataset

Информационно-аналитическое приложение для анализа пользовательских предпочтений в сфере кинематографа и формирования персонализированных рекомендаций фильмов.

## Архитектура приложения

Приложение состоит из трёх слоёв:

- **Backend** (`Scripts/`) — загрузка данных, операции над справочниками, формирование текстовых отчётов.
- **ML** (`Scripts/`) — рекомендательная система и генерация графических отчётов.
- **Frontend** (`Scripts/`) — графический интерфейс на Tcl/Tk, отображение отчётов и графиков, настройка приложения через `.ini`-файл.

Весь код написан в функциональном стиле (без ООП) и соответствует стандартам **PEP 8** и **PEP 257**. Все функции содержат docstring и комментарии.

### Сценарий первого запуска

1. Приложение автоматически загружает данные из MovieLens Dataset и формирует справочники в 3НФ.
2. Пользователь проходит регистрацию — в таблицу `Users` добавляется новая запись.
3. Пользователь указывает предпочтительные жанры и несколько ранее просмотренных фильмов.
4. На основе этих данных система начинает формировать персонализированные рекомендации.

### Сценарий повторного запуска

Приложение загружает справочники из бинарных файлов (pickle) в каталоге `Data/`, минуя повторную обработку CSV.

## Структура базы данных

База данных приведена к третьей нормальной форме и состоит из шести таблиц.

### Users
| Поле | Тип | Описание |
|------|-----|----------|
| `userId` | int, PK | Идентификатор пользователя |

### Movies
| Поле | Тип | Описание |
|------|-----|----------|
| `movieId` | int, PK | Идентификатор фильма |
| `movieName` | str | Название фильма |

### Genres
| Поле | Тип | Описание |
|------|-----|----------|
| `genreId` | int, PK | Идентификатор жанра |
| `genreName` | str | Название жанра |

### Movie-Genre
| Поле | Тип | Описание |
|------|-----|----------|
| `movieId` | int, FK → Movies | Идентификатор фильма |
| `genreId` | int, FK → Genres | Идентификатор жанра |

### Ratings
| Поле | Тип | Описание |
|------|-----|----------|
| `ratingId` | int, PK | Идентификатор оценки |
| `userId` | int, FK → Users | Идентификатор пользователя |
| `movieId` | int, FK → Movies | Идентификатор фильма |
| `rating` | float | Оценка фильма (0–5) |

### Tags
| Поле | Тип | Описание |
|------|-----|----------|
| `tagId` | int, PK | Идентификатор тега |
| `userId` | int, FK → Users | Идентификатор пользователя |
| `movieId` | int, FK → Movies | Идентификатор фильма |
| `tag` | str | Текстовый тег |

## API: Backend-модуль

### `load_data_csv(filepath)`

Читает CSV-файлы MovieLens из указанной папки и возвращает сырые данные.

**Параметры:** `filepath: str` — путь к папке `Data/`

**Возвращает:**
```python
{
    "movies":  pd.DataFrame,  # сырые данные о фильмах
    "ratings": pd.DataFrame,  # сырые данные об оценках
    "tags":    pd.DataFrame   # сырые данные о тегах
}
```

---

### `convert_to_3nf(raw)`

Преобразует сырые данные в набор таблиц третьей нормальной формы.

**Параметры:** `raw: dict` — результат `load_data_csv()`

**Возвращает:**
```python
{
    "users":       pd.DataFrame,
    "movies":      pd.DataFrame,
    "genres":      pd.DataFrame,
    "movie-genre": pd.DataFrame,
    "ratings":     pd.DataFrame,
    "tags":        pd.DataFrame
}
```

---

### `init_db(path)`

Инициализирует базу данных при запуске: загружает из pickle, если файлы существуют, иначе вызывает `load_data_csv` → `convert_to_3nf`.

**Параметры:** `path: str` — путь к папке `Data/`

**Возвращает:** `db: dict` — словарь с таблицами в 3НФ (аналогично `convert_to_3nf`)

---

### `save_db(df, filepath)`

Сохраняет `DataFrame` в бинарный файл формата pickle.

**Параметры:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `df` | `pd.DataFrame` | Таблица для сохранения |
| `filepath` | `str` | Путь к файлу (например, `Data/users.pkl`) |

**Возвращает:** `None`

---

### `load_db(filepath)`

Загружает `DataFrame` из бинарного файла pickle.

**Параметры:** `filepath: str` — путь к `.pkl`-файлу

**Возвращает:** `pd.DataFrame`

---

### Операции над справочниками

| Функция | Описание |
|---------|----------|
| `add_user(db)` | Добавляет нового пользователя |
| `delete_user(db, user_id)` | Удаляет пользователя и все его оценки и теги |
| `edit_user(db, user_id)` | Редактирует данные пользователя |
| `add_movie(db)` | Добавляет новый фильм |
| `delete_movie(db, movie_id)` | Удаляет фильм из справочника |
| `edit_movie(db, movie_id)` | Редактирует данные фильма |
| `add_rating(db, user_id, movie_id, rating)` | Добавляет оценку (0–5) |
| `delete_rating(db, rating_id)` | Удаляет оценку |

## Зависимости и запуск

### Установка зависимостей

Все зависимости перечислены в `requirements.txt`:

```bash
pip install -r requirements.txt
```

### Запуск приложения

```bash
python Scripts/main.py
```

### Используемые библиотеки

| Библиотека | Назначение |
|------------|------------|
| `pandas` | Работа с данными, формирование справочников и отчётов |
| `numpy` | Вычисления при формировании рекомендаций и статистик |
| `matplotlib` | Построение графических отчётов |
| `tkinter` | Графический интерфейс (входит в стандартную библиотеку Python) |

> Используются версии библиотек, входящие в состав дистрибутива **Anaconda**, актуального на дату начала работы над проектом. Приложение разработано для платформы **MS Windows 10**.
