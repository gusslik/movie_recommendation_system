## 3. Описание модулей

### 3.1. `recommendations.py`

Содержит функции для коллаборативной фильтрации.

| Функция | Описание |
|---------|----------|
| `build_user_item_matrix(ratings_df)` | Строит матрицу «пользователь — фильм» (строки — `userId`, столбцы — `movieId`, значения — оценки) |
| `cosine_similarity_matrix(user_item_matrix)` | Вычисляет косинусное сходство между всеми пользователями (нормализация, скалярное произведение). Возвращает квадратную матрицу сходства |
| `get_user_based_recommendations(...)` | User-based рекомендации: находит топ-N похожих пользователей, усредняет их высокие оценки (≥4), исключает уже оценённые фильмы. Если кандидатов нет — возвращает популярные фильмы |
| `build_item_item_similarity(user_item_matrix)` | Транспонирует матрицу и вызывает `cosine_similarity_matrix` для получения сходства между фильмами |
| `get_item_based_recommendations(...)` | Item-based рекомендации: для каждого фильма с оценкой ≥4 находит похожие фильмы и суммирует оценки с весами |
| `get_popular_movies(ratings_df, movies_df, top_n)` | Возвращает топ-N фильмов с наибольшим количеством оценок (используется как fallback) |
| `user_similarity_matrix(ratings_matrix)` | Обёртка для `cosine_similarity_matrix` (единый интерфейс) |
| `item_similarity_matrix(ratings_matrix)` | Обёртка для `build_item_item_similarity` |
| `get_recommendations(user_id, top_n=5, method='user')` | Главная функция для вызова из GUI. Сама загружает данные, строит матрицу и вызывает user- или item-based метод |

### 3.2. `plot_reports.py`

Функции для построения графиков (все возвращают `None`, но сохраняют файл или показывают окно).

| Функция | Тип графика | Требуемые колонки |
|---------|-------------|-------------------|
| `clustered_bar(df, cat_col1, cat_col2, ...)` | Кластеризованная столбчатая диаграмма | два категориальных |
| `categorized_hist(df, quant_col, cat_col, ...)` | Категоризированная гистограмма | количественный + категориальный |
| `categorized_boxplot(df, quant_col, cat_col, ...)` | Box-Whisker | количественный + категориальный |
| `categorized_scatter(df, x_col, y_col, cat_col, ...)` | Диаграмма рассеяния с цветом по категории | два количественных + категориальный |

Все функции поддерживают:

- Параметр `save_path` — если передан, график сохраняется в указанный файл (PNG), иначе отображается интерактивно.
- Автоматическое создание папки для сохранения (`os.makedirs`).
- Проверку наличия колонок и достаточности данных — при ошибках выводят сообщение в консоль и завершаются без исключения.

## 4. Алгоритм рекомендательной системы

### 4.1. Матрица оценок и косинусное сходство

- Таблица `ratings` (`userId`, `movieId`, `rating`) преобразуется в матрицу R размером U × M (пользователи × фильмы) с помощью `pivot_table`. Пропуски (NaN) — фильмы, не оценённые пользователем.
- Для косинусного сходства пропуски заменяются на 0.
- Нормализация строк: каждая строка делится на свою евклидову норму. Это делает векторы единичными, чтобы сходство зависело только от направления, а не от активности пользователя.
- Косинусное сходство между пользователями i и j вычисляется как скалярное произведение нормализованных строк:

```
similarity[i, j] = R_norm[i] · R_norm[j]
```

Результат: квадратная матрица, где на диагонали 1, значения от 0 до 1.

### 4.2. User-based рекомендации

Для пользователя `u`:

1. Из матрицы сходства берутся все другие пользователи с положительным сходством, сортируются по убыванию — отбираются топ-20.
2. Для каждого такого пользователя `v` выбираются фильмы, которые он оценил на 4 или 5 и которые `u` ещё не оценил.
3. Для каждого фильма-кандидата накапливается взвешенная сумма:
   ```
   score[m] += similarity(u, v) * rating(v, m)
   ```
4. Фильмы сортируются по убыванию `score`, выбирается `top_n`. Если кандидатов нет — возвращаются самые популярные фильмы.

### 4.3. Item-based рекомендации

1. Матрица R транспонируется (строки — фильмы, столбцы — пользователи), вычисляется косинусное сходство между фильмами.
2. Для пользователя `u` определяются фильмы, оценённые на ≥4.
3. Для каждого такого фильма `m` находятся топ-10 похожих фильмов (исключая сам `m` и уже оценённые). Для каждого похожего фильма `n` накапливается:
   ```
   score[n] += similarity(m, n) * rating(u, m)
   ```
4. Сортировка и выбор `top_n`. Если кандидатов нет — возвращаются популярные фильмы.

### 4.4. Fallback — популярные фильмы

Функция `get_popular_movies` считает количество оценок для каждого фильма, сортирует по убыванию и возвращает `top_n` с `score = 0`. Это обеспечивает гарантированный ответ даже при отсутствии персональных рекомендаций.

## 5. Графические отчёты

Все графики строятся с использованием `matplotlib.pyplot`. Перед сохранением создаются недостающие папки.

### 5.1. Кластеризованная столбчатая диаграмма

```python
clustered_bar(users, 'gender', 'occupation', save_path='Graphics/bar.png')
```

Строит таблицу сопряжённости `pd.crosstab(df[cat_col1], df[cat_col2])` и рисует её методом `plot(kind='bar')`. По оси X — уникальные значения `cat_col1`, столбцы разбиты на подгруппы по `cat_col2`.

### 5.2. Категоризированная гистограмма

```python
categorized_hist(users, 'age', 'gender', bins=15, save_path='Graphics/hist.png')
```

Для каждого значения категориальной переменной рисует гистограмму количественной переменной с прозрачностью `alpha=0.5`. Легенда показывает категории.

### 5.3. Box-Whisker

```python
categorized_boxplot(users, 'age', 'occupation', save_path='Graphics/box.png')
```

Использует `df.boxplot(column=quant_col, by=cat_col)`. Для каждой категории рисуется отдельный ящик с усами.

### 5.4. Диаграмма рассеивания

```python
categorized_scatter(user_avg, 'age', 'avg_rating', 'gender', save_path='Graphics/scatter.png')
```

Для каждой категории из `cat_col` рисует точки (`x_col`, `y_col`) разным цветом (палитра `tab10`). Точки полупрозрачные (`alpha=0.6`). Перед построением удаляются строки с NaN, проверяется наличие колонок и минимальное количество данных.

## 6. Интеграция с GUI

Фронтенд-разработчик может вызывать следующие функции:

**Рекомендации:**

```python
get_recommendations(user_id=1, top_n=5, method='user')
```

**Графики:**

Любая функция из `plot_reports` принимает `save_path`. Если нужно отобразить график внутри окна Tkinter:

- Можно не передавать `save_path` — тогда вызовется `plt.show()`.
- Для встраивания рекомендуется передавать `save_path` во временный файл, а затем загружать его через `PhotoImage` или использовать `FigureCanvasTkAgg` после небольшой доработки функции.
